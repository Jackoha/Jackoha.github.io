前言

最近剛好把 cryptohack 的題目刷完了
而且也很長一段時間沒有寫 blog 了 最近好懶
這篇雖然叫做 “如何破台 cryptohack” 但應該不會提到太多技術細節
反而是怎麼去查資料解題的感覺

起因

cryptohack 算是我剛接觸資安就有在碰的一個練習平台
在那個 LLM 打 crypto 沒那麼強的時代，想練 crypto 這就是最好的平台了
那時看到榜上有 maple, whale 一些超猛的 crypto 佬
讓我產生了一定的憧憬，讓我也想要努力花時間爬到前面的名次
但大家也都知道 crypto 是一個需要算數學 + 寫一堆 code 的領域
那時的我寫 code 既不強，也沒有強韌的數論基礎，遇到題目就只能 cry 了
因此我常常會有解題時間的大斷層，常常隔一個月或好幾個月才繼續解題

ECC~lattice

當我憑藉著我糟糕的數學能力想辦法撐過 RSA 跟 diffle-hellman 後
我接下來便被緊接而來的橢圓曲線跟格密碼給爆揍一頓
開始出現各式各樣抽象的算法
我的腦袋已經慢慢跟不上這些複雜的數學運算以及知識了
因此我在這時做了一個相當重要的決定，也就是不要完全理解他
這個時候你可能會問 “不理解這些東西，要怎麼面對題目呢”
那我舉個極端一點的例子吧
你今天是一個打 web 的新手，你不會為了打一個簡單的 web 題而想著 “我是不是該去逆向 http server”
比較重要的是知道這些算法要用在哪些地方跟怎麼用，如果為了一個算法去死命研究他所有原理，我認為是不值得的
但我不否認你成功後會超強就是了
在打這一部份的時候我也是參考了很多資料跟 write-up 才有辦法順利進行
這邊就丟一些ㄅ

  • Elliptic Curve
  • Crypto Book
  • 我打 cryptohack 時找的提示

但就算不找這些也能從題目的名稱或他給的參考資源找到一些線索，有耐心的話這些題目都是不難的

如果有看到我打 cryptohack 的時間軸應該會發現九月一直狂刷分，因為我想在特選前拿到台灣前十

衝刺期

特選期間就沒怎麼打 cryptohack 了，這段期間 crypto 的進步都是在平常打 CTF 的題目跟其他人的 write-up
加上這個時候開始有人會用 LLM, agent 等工具打 crypto
變成一種有人無腦丟就可以達到花時間看題目的人的程度，讓我對 crypto 有點難過
總覺得不太公平，有點失去當初打 crypto 那種花時間動腦的樂趣了
不過就在某一天我突然想起這個平台想說去看個 leaderboard
whale 怎麼突然就一萬三千分了 這讓我大為震驚
這個時候我沒什麼目標，過著沒事就去打瓦的日子
於是我就想說 “不然我每天打 cryptohack 打到超過 whale 好了”
接著就是從 3/6 一路爆衝一個多月
從原本的分數要超過 whale 變成了要把 cryptohack 破台
這段期間的解題過程也比我想像中的順利很多，可能是平常打 CTF 累積下來的經驗吧
當然可能有人會問我這些東西你真的理解嗎
我可以很肯定地回答你 我的腦袋已經理解不了這些了
大概從 Isogeny 開始，我都是用拼拼湊湊的方式想辦法解題
我也只是理解他的概念，如果要我解釋數學的部分我只能說想都別想
但因為他都有給一些參考資料所以我解的還算順利

不過既然我都這樣講了就代表大家其實也都不需要這樣就能解很多題目了
如果你真的不行也可以問問 gemini，他真的超會算數學
總而言之，給大家一些 cryptohack 難題的建議就是

  • 去研究題目給的 resource
  • 翻找各種能幫你處理算法的腳本
  • 耐心的思考
    相信大家會覺得上面這三點很沒用，但我實際感覺下來就是這樣子
    當然也有可能是我特選完時間很多
    但有認真花時間思考還是能感受到其中的有趣之處的

結語

其實我在想現在願意打 crypto 甚至是打 CTF 是不是越來越少了
那些很猛的模型 agent 都能把大部分題目打遍是不是讓大家對 CTF 意願越來越低了
當然我自己並不是說用 agent 不好，現在誰不用 agent
不過我也感受到這樣的狀況就是讓出題者使出自己藏最深的招數
或許現在才是 CTF 最黃金的時期也說不定?
但拜託不要讓我在看一些 code base 超大的抽象東西了
